一、 会议背景与核心目标

本次会议旨在深入讨论并敲定由微尘提出的“关于引入核心ID作为正式统计与治理唯一身份标识的建议”的提案。核心目标是解决当前因参与者使用多个昵称或外部身份导致的数据统计去重困难、不一致等问题，为未来建立自动化、可审计的治理系统奠定技术基础。

二、 核心议题讨论与决议

讨论主要围绕以下几个议题展开，并形成了相应决议：

议题一：core ID的定位与核心原则

提案内容（微尘）：

o定位： core ID应作为系统内唯一的、标准化的身份标识，是所有正式统计、审计记录的唯一主键。

o核心原则： 实行“身份标识”与“社交展示”相分离。core ID用于机器识别和核心数据统计，昵称则保留在社交层用于日常交流，不参与量化计算。

o价值： 提升数据的可追溯性、可审计性，为智能化分析和AI辅助治理提供干净、标准化的数据基础。

讨论与决议：

o全体与会人员一致认可该提案的必要性和系统性。

o如行 补充指出，唯一ID能有效解决用户地址和域名可变带来的溯源难题。

o决议： 正式确立core ID作为社区统一身份标识的核心地位。

议题二：core ID的标准化落地方案与执行细节

o执行原则： 在推行标准化的同时，不影响社区的交流自由。

o应用场景： 所有需要正式留痕的工作记录，包括但不限于：

1.会议发言与纪要

2.GitHub提交、合并请求

3.公众号稿件署名

4.工作量统计表格

o署名格式讨论：

1.微尘 最初主张在统计和正式记录中仅使用core ID，以确保数据的纯粹性和机器可读性。

2.楼兰渔夫（悟空） 与 Bataroc提出应兼顾人性化与亲和力，建议采用 “昵称 + core ID” 的组合格式进行展示。

3.最终共识： 在数据统计、后台处理等核心环节，强制且仅使用core ID；在前端展示（如会议纪要署名、社区交流）时，鼓励使用“昵称 + core ID”的格式，以实现标准化与个性化的平衡。

o过渡期安排： 针对12月及以前的历史数据，设定过渡期进行处理和映射，确保数据的连贯性。

议题三：core ID的延伸思考与潜在问题

o1. 与链上治理的结合：

1.教链 提出前瞻性构想：未来可将经过KYC认证的core ID直接作为链上投票权的基础，替代额外发行投票权代币的复杂方案，实现“一人一票”并简化治理模型，有效避免女巫攻击。

2.讨论焦点： 该方案的优势在于简化流程，但需确保KYC的质量与可靠性，并制定针对不活跃（僵尸）账号的管理规则。

o2. 身份注销与历史数据留存：

1.问题： 用户注销core ID后，其对应的链上签到记录（如NFT）可能在前端无法显示，影响历史贡献的追溯。

2.解决方案： 教链 建议，通过链上查询技术手段，依然可以获取历史记录。同时，可在系统底层的“基本信息表”中保留历史签约信息，以确保数据完整性与可验证性。会议确认了技术上的可行性。

o3. 技术细节确认：

1.core ID格式： 确认为纯数字编号（如0001, 0099），机器识别稳定。会议指出需保持编号规则的一致性，避免未来因位数扩展引发问题。

2.系统容量： 确认链上ID容量近乎无限，无需担心耗尽问题。

3.唯一性： 注销的core ID编号将保留，不可复用，以保证身份的绝对唯一性。

三、 会议总结与待办事项

会议总结： 本次会议成功通过了引入core ID的提案，明确了其作为核心基础设施的战略意义。会议在“强制标准”与“柔性展示”之间找到了平衡点，并对其长期应用于链上治理和数据管理进行了富有建设性的探讨。
