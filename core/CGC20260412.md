##CGC会议20260412讨论区

教链(J-25)在 Jouleverse CGC 会议（2026.04.12）上的发言纪要

一、对统计表格优化提案的态度

教链明确表示支持Bataroc和微尘提出的统计表格合并及迁移至飞书的改进方案，认为这是非常重要的一步，尤其肯定了将表格从 Excel 迁移到飞书的做法。

二、对未来发展的战略建议

教链指出，飞书表格的优化不仅服务于当前手工统计的高效便捷，更应着眼于与 AI 智能体（Agent） 的深度结合。

他提出以下几点：

Agent 友好的方向：

表格和统计流程应逐步调整为对智能体友好，未来统计工作可通过智能体自动完成，如：

成员通过语音或文字指令让智能体自动汇总 GitHub 数据并提交至飞书表格；

统计主管只需下达指令，智能体即可自动生成汇总、转换格式并提交至 GitHub。

MCP 标准的重要性：

教链介绍了 Anthropic 提出的 MCP 标准，并指出淘宝等平台已开始支持该标准。他认为，

Jouleverse 的链上数据、GitHub、飞书等也应通过 MCP 接口实现智能体友好化，从而让智能体串联起链上链下工作流。

工作流的未来构想：

理想状态下，人只需下达指令，智能体即可完成从数据抓取、汇总、审核到产出物生成和提交的全流程。

这将极大解放人力，使团队专注于更高价值的任务。

三、总结性观点

教链认为，当前统计表格的优化是“将火车放到高速铁路上”，方向正确，后续应持续迭代，最终实现统计工作的 

自动化和智能化，为 Jouleverse 的整体发展赋能。

楼兰渔夫(J-4)在 Jouleverse CGC 会议（2026.04.12）上的发言纪要：

我个人是很赞成这种优化的我们其实最大的不确定性跟会影响整个数据，这个 core 成员的各种贡献的这方面就是

统计这一块的数据的一个复杂性跟滞后性而带来的问题。这方面如果能得到一个规范的处理，应该提高效率，

对整体项目是加分的赋能的，我觉得是对的。
