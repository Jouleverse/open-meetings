 ### J-25:教链发言要点总结

### 一、AI 助理（机器人）的应用与协作

1. **当前工作内容**
   - 利用 AI 助理（如"小新"）结合 Jouleverse 工作流，进行文档 Review、更新 Core 成员列表、Fork 仓库、修改并提交 PR。
   - 修复了水龙头（领 Gas）功能。（by 大白助理）
   - 尝试让 AI 助理（小新）根据指令撰写节点搭建文档，并进行人工 Review 后发布。

2. **协作方式**
   - 鼓励通过 GitHub PR 进行文档/规则的修订与讨论，替代纯口头或微信群讨论，实现异步协作。
   - PR 是开源世界的标准贡献方式，便于公开讨论、方案竞争和共识形成。
   - 建议将工作流程规范化，如通过 PR 提出修订方案，经过评论、修改后由仓库维护者集中决策合并。

3. **AI 使用策略**
   - 对于通用知识，AI 可直接执行；对于项目专有知识（如 Jouleverse 的 Gas 规则），需明确告知并补充背景资料。
   - 提倡"带着 AI 做一遍，再让它自己总结成技能文档"的方法，比纯人工编写说明书更高效。
   - 人的角色应从"执行者"转向"审核者"，重点放在检查和反馈环节（PDCA 循环）。

### 二、激励发放与多签人轮值

1. **激励发放执行**
   - 支持尽快执行一次激励发放，以验证统计流程的完整性，便于后续自动化统计脚本的开发。
   - 建议多签人（执行发放角色）采用固定轮值顺序，避免依赖单一人员造成系统性风险。

2. **多签人变更规则**
   - 认可微尘提出的"多签人变更当月激励归属"问题，建议通过 PR 方式正式修订规则，形成书面规范。

### 三、技术工具与成本建议

- 针对 Koant 的龙虾助理因欠费停用，建议改用 DeepSeek 官方按量计费 API（更经济），并可由教链的机器人协助远程配置。
- 提到腾讯的 WorkBuddy 可作为本地 AI 助手，免费且适合不熟悉服务器配置的用户。

### 四、治理与协作理念

- 强调**民主集中制**在开源项目中的适用性：民主讨论，集中决策（由仓库维护者最终合并）。
- 反对"多数人暴政"式的简单投票，专业方向应基于讨论质量而非人数。
- 提倡使用 PR 作为议案提出和竞争的工具，实现人人可提案的"技术平权"。

### 五、对新人刘桂洪的回应

- 肯定微尘的建议，鼓励刘桂洪将使用 WorkBuddy 搭建节点的实际指令过程整理成稿，分享经验，因为"很多人还不会指挥 AI 干活"。
- 强调"听说、想到、做到"之间存在差距，实操经验很有价值。
